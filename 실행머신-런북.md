# 노무 리서치 포털 — 실행 머신 런북

상시 켜둔 머신(Claude Code 설치·MCP 인증 완료)에서 이 절차대로 돌립니다.

## 0. 준비물

| 항목 | 값 |
|---|---|
| Apps Script URL | config.js 와 동일한 `.../exec` |
| SECRET | Code.gs 의 `SECRET` 과 동일한 문자열 |
| 포털 repo 경로 | GitHub Pages 로 서비스되는 로컬 clone 폴더 |
| MCP 인증 | korean-law(claude.ai 커넥터) + NotebookLM 로그인 유지 |

> ⚠ MCP 주의: korean-law 는 원격 커넥터라 세션에 따라 도구가 빠질 수 있습니다.
> 실행 전 `/mcp` 로 연결 확인, NotebookLM 은 `nlm login` 후 `notebook_list` 로 실검증.

## 1. 새 질의 가져오기 (폴링)

```powershell
# 대기(recv) 상태 질의만 JSON 으로 받아오기
$SECRET = "여기에_SECRET"
$URL = "여기에_APPS_SCRIPT_URL"
Invoke-RestMethod -Uri "$URL`?action=pending&key=$SECRET" -Method Get
```

각 항목: `id, name, dept, depth, question` 을 받습니다.

## 2. 리서치 실행 — depth 별 프로파일

가져온 각 질의를 Claude Code 로 아래 깊이에 맞춰 실행합니다.

- **full (풀 검증)** — `labor-ground-truth`(법제처 원문) → `counter-advocate`(반대결론 변론) → `citation-auditor`(인용 재검증). 민감·중요 사안.
- **standard (표준)** — `labor-ground-truth` + 인용검증 1회.
- **quick (빠른 초안)** — `labor-ground-truth` 로 핵심 조문·해석 위주.

출력물은 **기존 규칙 그대로**: 질문 포함 + 결론(안 되면 우회 / 되면 실행 도식) + **팝업 각주 HTML**.
링크는 공개 URL만(판례 law.go.kr precSeq, 예규 taxlaw ntstDcmId), DRF OC 링크 금지.

## 3. 답변 발행 (GitHub Pages)

1. 완성된 답변 HTML 을 `answers/<id>.html` 로 저장
2. `answers/index.json` 의 `items` 에 아래 형식으로 추가/갱신

```json
{
  "id": "NR-260701-005",
  "ts": 1751300000000,
  "date": "2026-07-01 14:30",
  "name": "홍길동",
  "dept": "인사팀",
  "question": "수습 3개월 직원 해고예고수당...",
  "status": "done",
  "answer_url": "answers/NR-260701-005.html"
}
```

3. commit & push → Pages 자동 반영

## 4. 시트 상태 되돌려쓰기

```powershell
$body = @{ action="update"; key=$SECRET; id="NR-260701-005"; status="done"; answer_url="answers/NR-260701-005.html"; worker="mainPC" } | ConvertTo-Json
Invoke-RestMethod -Uri $URL -Method Post -ContentType "text/plain" -Body $body
```

상태 값: `recv`(접수) → `proc`(처리중) → `done`(완료) / `error`(오류)

## 5. 반자동 루프

Claude Code 안에서:

```
/loop 10m 새 노무질의 폴링→depth대로 리서치→answers 발행→시트 done 갱신
```

- 10분마다 대기열 확인 → 있으면 실행/발행, 없으면 넘어감.
- 사장님은 감독만. 민감 사안(full)은 발송 전 육안 확인 권장.
